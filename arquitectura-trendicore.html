⚡ OVERVIEW

Visión General & Stack Tecnológico

Trendicore es una plataforma de inteligencia de tendencias que combina selección humana experta con inteligencia artificial. La web funciona como un marketplace curado de productos trending con un modelo B2B para marcas.

Stack Principal

▲

Next.js 16.1.6

Framework React con Turbopack, App Router, Server/Client Components y API Routes.

⚛

React 19.2.3

Última versión estable con Hooks, Context API, Suspense y Server Components.

◆

Supabase

Backend-as-a-Service: PostgreSQL, Auth, Storage y Real-time. Base de datos y autenticación.

💳

Stripe

Pasarela de pagos para suscripciones premium y checkout sessions.

✦

Framer Motion

Animaciones declarativas con LazyMotion para rendimiento óptimo.

✉

Resend

Servicio de email transaccional para newsletters y comunicaciones.

🎯 CLEAN ARCH

Clean Architecture & DDD

El directorio src/core/ implementa Clean Architecture con Domain-Driven Design, separando la lógica de negocio del framework y la infraestructura.

PRESENTACIÓN components/ + app/ — React Components, Pages, Layouts. Dependen de Application.
APLICACIÓN core/*/application/ — Use Cases: CreateProductUseCase, GetProductsUseCase, TrackClickUseCase, etc.
DOMINIO core/*/domain/ — Entidades (Product), Errors (ProductErrors), Interfaces (IProductRepository).
INFRAESTRUCTURA core/*/infrastructure/ — Adaptadores Supabase, servicios HTTP, storage. Implementan las interfaces del dominio.

Flujo de Datos

🖥 UI Component → 🪝 Custom Hook → 📁 API Route → ⚙️ Use Case → 🔌 Repository → 🗄 Supabase

Dominios Implementados

Dominio Entidades Use Cases Responsabilidad
Product Product, ProductErrors Create, Get, Update, Delete, TrackClick CRUD de productos, tracking de clics afiliados, validaciones de negocio
User User profile Auth, Profile Autenticación, perfiles, roles
Storage Image assets Upload, Delete Gestión de imágenes de producto (Supabase Storage)
Shared Domain errors — Errores y utilidades compartidas entre dominios
🎨 DESIGN SYSTEM

Design System

El design system está definido en globals.css y usa CSS Custom Properties (variables CSS) para mantener consistencia. El estilo general es editorial premium con temas light/dark automáticos.

Filosofía de Diseño

🏛 Editorial Premium

Bordes finos (1px solid var(--border)), sombras sutiles y arquitectónicas, radios contenidos, tipografía con tracking negativo. Inspiración en revistas de moda y plataformas SaaS.

🌗 Dual Theme

Light mode con fondo crema (#F9F9F9) y dark mode profundo (#0A0A0A). Cambio automático via data-theme attribute con script anti-flash en <head>.

📐 CSS Modules

Cada componente tiene su .module.css para encapsulación total de estilos. No hay utilidades globales tipo Tailwind — todo es CSS vanilla scoped.

✨ Vibrant Accents

Acentos vibrantes (púrpura, rosa, cian) sobre fondos neutros. Gradientes animados en el hero y scrollbar. Glows sutiles en hover.

🔤 TIPOGRAFÍA

Tipografía

Las fuentes se cargan con next/font/google y se inyectan como CSS variables en el <body>. No hay flash de fuentes (FOUT).

Plus Jakarta Sans

Variable: --font-sans

Uso: Cuerpo de texto, párrafos, botones, labels, navegación.

Estilo: Geométrica, moderna, altamente legible. Pesos: 400–900.

Outfit

Variable: --font-display

Uso: Títulos, hero, sección headers, estadísticas, precios.

Estilo: Display sans-serif con personalidad. Pesos: 400–900.

🧩 COMPONENTES

Componentes UI

Los componentes están organizados por dominio: ui/ para interfaz global, product/ para catálogo, admin/ para dashboard, y providers/ para gestión de estado global.

Componentes de Interfaz (ui/)

Navbar

Sticky, backdrop-blur, logo + links + actions. Hamburger menu para mobile con animación slide-down. Incluye login/avatar, wishlist badge y theme toggle.

Footer

4 columnas: marca + redes, navegación, legal, y newsletter form. Incluye disclaimer de afiliados y copyright dinámico.

TrendingBar

Barra horizontal con tags trending que enlazan a categorías. Scroll horizontal en mobile. Debajo del Navbar.

Button

Componente reutilizable con variantes (primary, secondary), tamaños, iconos izquierda/derecha. Soporta href para Links.

SearchBar

Input de búsqueda con icono, bordes sutiles y transición focus. Filtra productos en tiempo real.

LocaleSelector

Selector de idioma (ES/EN) con dropdown animado. Persiste en cookie trendicore-locale.

RegionSelector

Selector de región para personalizar productos según país. Controla filtros de catálogo por región.

ThemeProvider

Context + toggle light/dark. Usa localStorage para persistir y script en <head> para evitar flash.

NoiseOverlay

Overlay con textura de ruido sutil sobre toda la página. Efecto papel/grano para profundidad visual.

Componentes de Producto (product/)

ProductCard

Card con imagen 3:4, hover overlay con CTAs (comprar + quick view), TrendBadge, botones de favorito/compartir, y mini CTA en la info section.

QuickViewModal

Modal de vista rápida con imagen grande, detalles del producto, trend score, y botón CTA directo a la tienda.

TrendBadge

Badge visual que muestra el Trend Score (🔥 1–100). Colores dinámicos según nivel: bajo (gris) → medio (amarillo) → alto (rosa/fuego).

FavoritesDrawer

Panel lateral tipo drawer que muestra la wishlist del usuario. Slide-in desde la derecha con backdrop oscuro.

ProductErrorBoundary

Error boundary React que captura errores de renderizado en las cards y muestra un fallback en lugar de romper la página.

Componentes Admin (admin/)

ProductForm

Formulario completo de creación/edición de producto con react-hook-form + zod. Upload de imagen, categorías, trend score, fuente de afiliado.

ProductList

Tabla/lista de productos con acciones (editar, eliminar, toggle featured). Soporta filtros y bulk actions.

DashboardCharts

Gráficos con Recharts: tendencias de clics, productos populares, métricas de rendimiento.

🔗 PROVIDERS

Context Providers

La gestión de estado global se realiza con React Context API. Los providers envuelven el layout público en un orden específico de dependencias.

Cadena de Providers

<LazyMotion> — Framer Motion (carga lazy de animaciones)
<ThemeProvider> — Gestión de tema light/dark
<LocaleProvider> — Idioma (es/en) + función t() de traducción
<AuthProvider> — Autenticación Supabase + user + profile
<WishlistProvider> — Wishlist con persistencia localStorage
<Navbar /> + <TrendingBar />
{children} + <Footer /> + <CookieBanner />

Hooks Públicos

Hook Provider Retorna
useTheme() ThemeProvider { theme, toggleTheme }
useLocale() LocaleProvider { locale, region, t, setLocale, setRegion }
useAuth() AuthProvider { isAuthenticated, user, profile, signOut }
useWishlist() WishlistProvider { items, count, isInWishlist, toggleWishlist }
useProducts(query) — (standalone) { products, loading, error } — Fetches desde API con memoización
🎭 ANIMACIONES

Animaciones & Microinteracciones

Las animaciones combinan CSS keyframes para efectos ambientales y Framer Motion para interacciones inteligentes (viewport-aware, stagger, springs).

Keyframes CSS Globales

gradient-shift

Anima el fondo del body con gradientes radiales sutiles que se mueven lentamente (20s, infinite alternate). Crea un efecto de ambiente vivo.

float

translateY suave de 15px arriba/abajo. Se usa en emojis y estados vacíos para efecto de levitación.

pulse-glow

Pulso de opacidad 0.4→0.8 para el dot verde del badge "Live". Efecto de "en línea".

shimmer

Loading skeleton: gradiente que se desplaza horizontalmente. Efecto de contenido cargando premium.

shine

Gradiente del hero highlight (rosa→púrpura→cyan) que se desplaza. Texto que brilla en ciclo de 5s.

spin-slow

Rotación 360° lenta para elementos decorativos. No se usa activamente pero disponible en el sistema.

Framer Motion — Patrones

// Entrada del Hero (fade-up con ease premium)
<m.div
  initial={{ opacity: 0, y: 40 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.7, ease: [0.22, 1, 0.36, 1] }}
/>

// Cards con scroll-triggered + stagger
<m.article
  initial={{ opacity: 0, y: 30 }}
  whileInView={{ opacity: 1, y: 0 }}
  viewport={{ once: true, margin: '-50px' }}
  transition={{ duration: 0.4, delay: index * 0.06 }}
/>

// Steps con stagger progresivo
transition={{ delay: i * 0.15, duration: 0.6 }}

📱 RESPONSIVE

Estrategia Responsive

La estrategia responsive usa mobile-first refinements con 3 breakpoints principales. El layout pasa de grids multi-columna a stacks verticales progresivamente.

Breakpoints

≤ 1024px — Tablet

Hero padding reducido, secciones más compactas. Grids mantienen columnas pero con menos gap.

≤ 768px — Mobile

Navbar → hamburger menu. Grids → 1-2 columnas. CTAs → stack vertical full-width. Product overlay deshabilitado (tap directo).

≤ 480px — Small

Tipografía reducida. Hero mínimo (auto height). Grid 2 columnas tight. Stats en wrap. Touch targets ≥ 44px.

Adaptaciones Clave

Componente Desktop Mobile (≤768px)
Navbar Links horizontales + actions Hamburger → slide-down panel con backdrop
Product Grid Masonry 4 columnas (CSS columns) 2 columnas, gap reducido
ProductCard Overlay Hover overlay con CTAs display: none — tap en card navega directo
Hero CTAs Flexbox horizontal Stack vertical, full-width, max-width: 320px
Steps Grid 3 columnas 1 columna, max-width: 480px centrado
Metrics Grid 4 columnas 2 columnas
Footer 4 columnas Stack vertical
🔌 API

API Routes & Backend

El backend está implementado con Next.js API Routes (Route Handlers). Cada endpoint sigue la convención de archivo route.js dentro de su directorio.

Endpoint Métodos Descripción
/api/products GET, POST, PUT, DELETE CRUD de productos. GET soporta filtros (category, region, featured, source)
/api/track-click GET Registra clic de afiliado y redirige al producto. Tracking de conversión.
/api/checkout_sessions POST Crea sesión de checkout Stripe para suscripciones premium
/api/webhooks POST Webhooks de Stripe para confirmar pagos y activar suscripciones
/api/newsletter POST Suscripción a newsletter. Guarda email en Supabase + envía bienvenida via Resend.
/api/admin/* Varios Endpoints protegidos para gestión administrativa
/api/v1/* GET API pública B2B v1 para clientes empresariales. Trend intelligence data.
🌍 I18N

Internacionalización

La internacionalización se gestiona con archivos JSON de traducción y un LocaleProvider personalizado. El idioma se persiste en una cookie para SSR.

Español (es.json)

Idioma principal. ~32KB. Cubre toda la UI, incluyendo labels de categorías, CTAs, mensajes de error, textos legales y SEO.

Inglés (en.json)

Traducción completa. ~30KB. Paridad 1:1 con español. Incluye traducciones de categorías de producto.

Mecanismo

// En cualquier componente client:
const { t, locale, region } = useLocale();

// Acceso a traducciones con dot notation:
t('hero.badge')           // → "INTELLIGENCE EN DIRECTO"
t('hero.ctaFashion')      // → "Explorar Moda"
t('product.categoryLabels.tech')  // → "Tech & Gadgets"

// La cookie 'trendicore-locale' se lee en el Root Layout
// para setear el atributo lang del <html> en SSR
🔍 SEO

SEO & Rendimiento

La optimización SEO está integrada a nivel de framework con metadata exportada, JSON-LD estructurado y sitemap dinámico.

Metadata Framework

Cada página exporta un objeto metadata con title, description, keywords, OpenGraph y Twitter Cards. metadataBase apunta a https://trendicore.net.

JSON-LD

Componente <JsonLd /> inyecta datos estructurados (Schema.org) en el Root Layout para Rich Snippets en buscadores.

Sitemap Dinámico

sitemap.js genera el sitemap automáticamente incluyendo todas las rutas públicas y productos.

Imágenes Optimizadas

next/image con atributos fill, sizes responsive y lazy loading automático. Fallback SVG para errores.

Performance

Turbopack

Dev server con Turbopack para HMR instantáneo (~1.4s cold start).

LazyMotion

Framer Motion con domAnimation para cargar solo las features necesarias (~60% menos bundle).

Font Optimization

next/font/google con display: swap y subsetting automático. Zero layout shift.
