Aplicación de Arquitectura Hexagonal

Guía de refactorización y ejemplos de código para el core de Trendicore

1. Nueva Estructura de Carpetas Propuesta

Para aplicar el análisis, debemos mover la lógica de negocio fuera de los componentes de React y de las rutas de API. Se propone crear una carpeta src/core/ completamente agnóstica de Next.js.

2. Entidad de Dominio (Domain)

El núcleo de la aplicación. Contiene los datos y las reglas invariantes de un Producto. No sabe qué es una base de datos ni qué es JSON.

// src/core/product/domain/Product.js
export class Product {
  constructor({ id, title, category, affiliateLink, affiliateSource, region, featured, status, createdBy }) {
    this.id = id;
    this.title = title;
    this.category = category;
    this.affiliateLink = affiliateLink;
    this.affiliateSource = affiliateSource;
    this.region = region;
    this.featured = featured || false;
    this.status = status || 'draft';
    this.createdBy = createdBy;
    
    this.validate();
  }

  // Reglas de negocio intrínsecas del modelo
  validate() {
    if (!this.title || this.title.trim().length === 0) {
      throw new Error('El producto debe tener un título válido');
    }
    if (!['amazon', 'shein'].includes(this.affiliateSource)) {
      throw new Error('Fuente de afiliado no soportada');
    }
    // Validar que la URL sea correcta (sin regex complicadas aquí para brevedad)
    if (!this.affiliateLink.startsWith('http')) {
      throw new Error('El enlace de afiliado debe ser una URL válida');
    }
  }

  // Comportamiento de dominio
  publish() {
    this.status = 'published';
  }
}

3. El Caso de Uso (Application)

Orquesta el flujo. Recibe datos (del controlador), instancia la entidad, y le dice al Repositorio (puerto) que guarde. Nótese que aquí inyectamos la dependencia (inversión de control).

// src/core/product/application/CreateProductUseCase.js
import { Product } from '../domain/Product.js';

export class CreateProductUseCase {
  // Inyectamos el puerto (interfaz del repositorio)
  constructor(productRepository) {
    this.productRepository = productRepository;
  }

  async execute(productData) {
    // 1. Instanciar y validar regla de negocio (arroja error si algo está mal)
    const product = new Product(productData);

    // 2. Comprobar si ya existe uno con el mismo enlace (Regla de aplicación)
    const existingProduct = await this.productRepository.findByLink(product.affiliateLink);
    if (existingProduct) {
      throw new Error('Ya existe un producto con este enlace de afiliado');
    }

    // 3. Persistir usando la abstracción, NO supabase directamente
    const savedProduct = await this.productRepository.save(product);
    
    return savedProduct;
  }
}

4. Adaptador Secundario (Infrastructure)

Esta es la única parte del sistema que conoce a Supabase. Implementa los métodos que el Caso de Uso necesita (save, findByLink).

// src/core/product/infrastructure/adapters/SupabaseProductRepository.js
// Este archivo adapta nuestro modelo de Dominio (Product) al modelo de BD (Filas de Supabase)

export class SupabaseProductRepository {
  constructor(supabaseClient) {
    this.client = supabaseClient;
  }

  async save(productEntity) {
    // Mapear de Dominio a Supabase (Data Transfer Object)
    const payload = {
      title: productEntity.title,
      category: productEntity.category,
      affiliate_link: productEntity.affiliateLink,
      affiliate_source: productEntity.affiliateSource,
      region: productEntity.region,
      featured: productEntity.featured,
      status: productEntity.status,
      created_by: productEntity.createdBy
      // Omitimos image_url por simplicidad aquí
    };

    const { data, error } = await this.client
      .from('products')
      .insert([payload])
      .select()
      .single();

    if (error) throw new Error(error.message);

    // Devolver una entidad de dominio reconstituida
    productEntity.id = data.id;
    return productEntity;
  }

  async findByLink(link) {
    const { data, error } = await this.client
      .from('products')
      .select('*')
      .eq('affiliate_link', link)
      .single();
      
    if (error && error.code !== 'PGRST116') throw error;
    return data ? true : null; // Retorna verdadero si existe
  }
}

5. Adaptador Primario (Framework - Next.js API)

La ruta de la API actúa como punto de entrada. Su única responsabilidad es recibir HTTP, inyectar el repositorio de Supabase en el Caso de Uso, y devolver respuestas HTTP.

// src/app/api/admin/products/route.js
import { NextResponse } from 'next/server';
import { supabaseAdmin } from '@/lib/supabase-admin';
import { CreateProductUseCase } from '@/core/product/application/CreateProductUseCase';
import { SupabaseProductRepository } from '@/core/product/infrastructure/adapters/SupabaseProductRepository';

export async function POST(request) {
  try {
    const body = await request.json();
    
    // Configuración de Inyección de Dependencias (Idealmente hecho en un Container)
    const repository = new SupabaseProductRepository(supabaseAdmin);
    const createProductUseCase = new CreateProductUseCase(repository);

    // Ejecución del Caso de Uso (La API no conoce lógica de negocio)
    const newProduct = await createProductUseCase.execute({
      title: body.title,
      category: body.category,
      affiliateLink: body.affiliate_link,
      affiliateSource: body.affiliate_source,
      region: body.region,
      featured: body.featured,
      status: body.status,
      createdBy: request.user.id // Asumiendo que obtienes esto del middleware
    });

    return NextResponse.json(newProduct, { status: 201 });
    
  } catch (error) {
    return NextResponse.json(
      { error: error.message }, 
      { status: error.message.includes('Ya existe') ? 409 : 400 }
    );
  }
}

Resumen

Con estos cambios:

  • Si cambias Next.js por Express, solo reescribes la Sección 5.
  • Si cambias Supabase por MongoDB, solo creas un MongoProductRepository (similar a la Sección 4) y lo inyectas en el caso de uso y el resto del programa no cambia.
  • Si un producto ahora requiere tener 3 imágenes obligatorias, añades la regla en Product.validate() (Sección 2) y automáticamente la API y la UI se ven afectadas y respetan la regla de negocio.
